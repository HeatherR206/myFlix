A web service for human users to search for and save information about movies.

OVERVIEW & ARCHITECTURE

The myFlix API is a RESTful API built with JavaScript/Node.js and the Express framework. It uses standard HTTP methods (GET, POST, PUT, and DELETE) to perform CRUD (Create, Read, Update, and Delete)operations on the available resources (endpoints) and returns a JSON response to the client.

Unlike "in-memory" data storage, which is temporary, this API uses MongoDB Community Server as its persistent NoSQL database. Mongoose, an Object Data Modeling (ODM) library, is used to interact with MongoDB, providing a structured, schema-based solution to model the application data.


üóíÔ∏è NOTE! All request and response bodies are encoded using syntax for either standard JSON or JSON Schema.

myFlix API executes with the following Node packages:

  • Express
  • Mongoose

  • Body-Parser middleware

Install Node packages via your system CLI

To install packages to your project's root folder:

  1. If necessary, read up on npm, Node's default package manager.

  1. To display the JSON data for your install, type the command: cat package.json

API TESTING TOOLS

Testing your URL endpoints is critical to developing a quality API. You will need to choose an HTTP Client Tool that can pinpoint errors in your API development. A few examples include Thunder Client, cURL, ReadyAPI, and Postman.

These tools will also be necessary to format data bodies for your HTTP requests (especially for POST), and responses (almost all).


HTTP METHODS

The myFlix API code gives users access to movie data via endpoints. When a client makes a request on a resource(s), the endpoints and HTTP methods direct the request to the appopriate back-end function, which is typically executed by a CRUD operation (CREATE, READ, UPDATE, and DELETE) on the database.

HTTP REQUEST METHOD CRUD OPERATION
POST CREATE
GET READ
PUT or PATCH UPDATE
DELETE DELETE

HTTP REQUESTS

HTTP Requests will follow Express syntax: app.METHOD(PATH, HANDLER)
For example: app.get('/movies', [handler]); will handle a request to the URL endpoint ‚Äú/movies" with the HTTP request method GET

app.delete('/users/:id', (req, res) => {
    const { id } = req.params;    

    let user = users.find( user => user.id == id );

HTTP STATUS CODES

The HTTP status codes provide high-level feedback about the outcome of client requests, as well as helping in debugging and handling errors successfully. All responses from the myFlix API will contain a status code.

‚úÖ Success Codes

Successful responses will return JSON data or a text-based message describing the transaction, and one of the following status codes:

HTTP Status Code Description
200 OK Request was successful
{ message: "User 2 has been deleted from database" }
201 Created New resource was successfully created

‚ö†Ô∏è Error Codes

Error responses will display a text-based message and one of the following status codes:

HTTP Status Code Description Example
400 Bad Request An error indicating the server does not understand the request due to a client-side error. { message: "No such movie exists in database" }
404 Not Found An error indicating the server cannot find the requested resource { message: "Sorry, that endpoint doesn't exist." }
500 Internal Server Error A generic error indicating the server has encountered a situation it cannot handle { message: "Uh-oh! Something broke" }
